기획팀 에이전트 상세 설계 v2 — 실데이터 기반

학습맵(XLSX)과 교육과정 표준체계(XLSX)를 입력받아, 학습맵 최소 depth 단위별 문항 메타정보를 자동 생성하는 하이브리드(규칙+LLM) 에이전트

학습맵 1,819 행
표준체계 419 건
교급 E · M · H
영역 4 개
성취기준 152 종
입력 데이터 분석 2 FILES

기획팀 에이전트의 입력은 2개의 XLSX 파일입니다. 학습맵은 문항 생성 단위(leaf node)를 결정하고, 표준체계는 성취기준·내용요소를 매핑합니다.

📌 표준체계ID 코드 구조: E4MATA01B03C05 → E(교급) + 4(학년군) + MAT(교과) + A01(영역:수와연산) + B03(1단계 내용요소) + C05(2단계 내용요소). 이 코드로 내용체계 영역, 내용요소, 성취기준을 한번에 역참조할 수 있습니다.
데이터 관계 N:M MAPPING

학습맵 1행은 여러 표준체계를 참조하고(N:M), 표준체계 1건도 여러 학습맵 행에 연결됩니다. 기획팀 에이전트는 이 관계를 풀어서 메타정보를 생성합니다.

🔗

연결 구조

RELATION // 학습맵 1행 → 다수의 표준체계ID 참조 (파이프 구분) 학습맵 행: "2022-2025-MAT-MAT-03-1-MN1-01-01-00-00" Depth1: "덧셈과 뺄셈" Depth2: "받아올림이 없는 세 자리 수의 덧셈을 어떻게 할까요" 표준체계ID: "E4MATA01B03C05" | "E4MATA01B03C07" ↓ JOIN ↓ JOIN ┌─────────────────────┐ ┌─────────────────────┐ │ E4MATA01B03C05 │ │ E4MATA01B03C07 │ │ 영역: 수와 연산 │ │ 영역: 수와 연산 │ │ 1단계: 세자리수 덧뺄 │ │ 1단계: 세자리수 덧뺄 │ │ 2단계: 덧셈의 계산원리│ │ 2단계: 덧셈의 계산 │ │ 성취: [4수01-03] │ │ 성취: [4수01-03] │ └─────────────────────┘ └─────────────────────┘
⚡ 에이전트 처리 방식: 학습맵 행당 연결된 표준체계를 모두 조회하여, ① 공통 성취기준 코드 추출 ② 내용체계 영역 결정 ③ 2단계 내용요소에서 핵심 개념 키워드 추출. 한 행에 평균 5~43개의 표준체계가 연결되어 있으므로, 중복 성취기준은 합산하지 않고 대표값을 선정합니다.

Leaf Node 판별 로직

LEAF NODE LOGIC // 최소 단위(leaf) 판별 — 문항 1건이 생성되는 단위 IF Depth3_단원명 IS NOT NULL: leaf_level = 3 leaf_name = Depth3_단원명 // e.g. "AI 익힘 문제" depth_path = "{Depth1} > {Depth2} > {Depth3}" ELIF Depth2_단원명 IS NOT NULL: leaf_level = 2 leaf_name = Depth2_단원명 // e.g. "받아올림이 없는 세 자리 수의 덧셈을 어떻게 할까요" depth_path = "{Depth1} > {Depth2}" ELSE: leaf_level = 1 leaf_name = Depth1_단원명 // e.g. "학기 초 AI 진단 평가" (26건) depth_path = "{Depth1}" // 결과: 1,819행 전체가 leaf 후보 // D2만 있는 행: 621건 — 주로 '학습 주제' 차시 단위 // D3까지 있는 행: 1,172건 — 주로 'AI 익힘 문제' 등 활동 단위
💡 활동유형에 따른 문항 성격 분류: Depth3의 단원명이 문항의 성격을 직접 결정합니다. "AI 익힘 문제"면 연습 문항, "대단원 도입"이면 진단 문항, "창의적으로 해결하기"면 심화 문항으로 분류됩니다. 이것이 규칙 엔진의 핵심 입력입니다.
처리 파이프라인 HYBRID

규칙 엔진이 학습맵 구조 + 표준체계 매핑으로 기본값을 결정하고, LLM이 성취기준 문맥을 분석하여 보정합니다.

STEP 1. XLSX 파싱 및 유효성 검증 규칙

두 파일을 파싱하고 필수 컬럼 존재, 데이터 타입, 교급 enum(E/M/H), 학습맵ID 형식 검증. 표준체계 테이블은 교육과정표준체계ID 기준으로 dict 인덱싱하여 O(1) 조회 준비.

🔍 검증 항목
  • 학습맵: 학습맵ID 유니크, 교급 ∈ {E, M, H}, 학년 범위(2~10), 표준체계 ID 비어있지 않은지
  • 표준체계: 교육과정표준체계ID 유니크, 내용체계 영역 ∈ 4개 영역, 성취기준 코드 패턴 검증
  • 교차 검증: 학습맵의 표준체계ID가 표준체계 테이블에 존재하는지 (413/520 매칭, 107건 누락 허용)

STEP 2. Leaf Node 확정 및 표준체계 JOIN 규칙

각 학습맵 행에서 leaf level(1/2/3)을 판별하고, 표준체계 ID 컬럼의 파이프(|) 구분 ID 목록을 파싱하여 표준체계 테이블과 JOIN합니다.

📦 JOIN 결과 집계
  • 내용체계 영역: 연결된 표준체계들의 영역을 집계 → 최빈 영역을 대표값으로 선정
  • 성취기준 코드: 중복 제거 후 배열로 수집 → 가장 많이 참조된 코드를 primary로 지정
  • 성취기준 내용: primary 코드의 기술문을 대표 성취기준으로 사용
  • 2단계 내용요소: 연결된 모든 2단계 내용요소에서 핵심 키워드를 추출

STEP 3. 규칙 기반 기본값 설정 규칙

3가지 축으로 기본값을 결정합니다: ① Depth3 활동유형 → 문항 성격, ② 내용체계 영역 → 문항 유형 후보군, ③ 교급+학년 → 난이도·배점 범위.

STEP 4. LLM 문맥 분석 및 보정 LLM

성취기준 내용 + 2단계 내용요소 + Depth2 학습 주제를 LLM에 전달하여, 규칙 기본값을 문맥 기반으로 보정합니다.

🤖 LLM 보정 영역
  • content_element 분류: 성취기준 기술문의 동사를 분석 → 지식이해(~이해하다, ~알 수 있다) / 과정기능(~할 수 있다, ~설명할 수 있다) / 가치태도(~인식하다, ~탐구하다)
  • 문항유형 세분화: '객관식' → '4지선다' vs '5지선다', '계산 원리' → 빈칸채우기 vs 서술형 판단
  • Bloom's Taxonomy: 성취기준 동사 분석 → 기억/이해/적용/분석/평가/창조
  • 출제 가이드: 2단계 내용요소를 기반으로 구체적 출제 방향·제약 조건 생성

STEP 5. 병합 및 충돌 해소 병합

규칙 기본값과 LLM 보정값을 우선순위에 따라 병합합니다.

⚖️ 우선순위
  • P1 불변: 학습맵ID, 표준체계ID[], depth_path, 교급, 학년, 학기, 내용체계 영역 — 원본 그대로
  • P2 LLM 우선: content_element, blooms_level, item_type_detail, authoring_guide, time_limit_sec
  • P3 범위검증: difficulty(규칙 허용범위 내면 LLM, 벗어나면 규칙), score(규칙 범위 내면 LLM)
  • P4 규칙 우선: item_type 대분류 — 활동유형 기반 규칙 우선, LLM은 세부유형만

STEP 6. JSON 출력 출력

학습맵 행당 1건의 메타정보 JSON 객체를 생성합니다. 전체 1,819건의 배열로 최종 출력.
