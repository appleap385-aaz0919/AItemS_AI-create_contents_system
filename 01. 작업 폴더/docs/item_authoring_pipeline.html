문항 저작 파이프라인

ITEM AUTHORING PIPELINE — 5 TEAM WORKFLOW

📥 학습맵 입력
TEAM 01
기획팀
학습맵 분석 → 문항 메타정보 생성

프로세스

01
학습맵 파싱
입력된 학습맵의 트리 구조를 분석하여 대분류 → 중분류 → 소분류 → 최소 단위(leaf node)까지 depth를 파악합니다.
02
최소 단위 추출
leaf node 기준으로 각 학습 요소를 식별하고, 해당 요소가 요구하는 학습 목표·성취기준을 매핑합니다.
03
문항 유형 결정
학습 목표에 적합한 문항 유형을 결정합니다 (객관식, 주관식, OX, 드래그앤드롭, 매칭, 서술형 등).
04
메타정보 스키마 생성
문항 ID, 난이도, 배점, 문항유형, 성취기준코드, 평가영역, 키워드, 소요시간 등을 포함한 JSON 메타데이터를 생성합니다.
산출물
item_meta.json
{ id, type, difficulty, score, achievement_code,
  domain, keywords[], time_limit, depth_path }
학습맵 분석 Bloom's Taxonomy 성취기준 매핑 메타데이터 스키마
TEAM 02
제작팀
메타정보 기반 → HTML 문항 저작

프로세스

01
메타정보 수신 및 해석
기획팀에서 전달된 JSON 메타정보를 파싱하여 문항 유형, 난이도, 평가 목표 등을 확인합니다.
02
문항 콘텐츠 설계
지문, 발문, 보기, 정답, 해설을 설계합니다. 난이도와 성취기준에 부합하는 내용을 구성합니다.
03
HTML 템플릿 매핑
문항 유형에 맞는 HTML 템플릿을 선택하고, 인터랙션(라디오, 체크박스, 드래그, 입력 등) 요소를 매핑합니다.
04
HTML 문항 저작
웹 표준 기반 HTML/CSS/JS로 문항을 구현합니다. 반응형, 접근성(A11y), 수식(MathML/LaTeX) 등을 고려합니다.
05
채점 로직 내장
정답 비교, 부분 점수, 서술형 키워드 매칭 등의 클라이언트 사이드 채점 로직을 구현합니다.
산출물
item_{id}.html
완전한 렌더링 가능 HTML 문항
(지문 + 발문 + 보기 + 인터랙션 + 채점로직)
HTML5 CSS3 Vanilla JS 반응형 접근성 MathML
TEAM 03
검수팀
오류 검증 · 정보 정확성 · 품질 검수

프로세스

01
메타정보 정합성 검증
기획팀이 정의한 메타정보(문항유형, 난이도, 성취기준)와 실제 제작된 문항의 내용이 일치하는지 검증합니다.
02
콘텐츠 정확성 검증
지문의 사실 관계, 정답의 정확성, 오답 보기의 타당성, 해설의 논리성을 검토합니다.
03
UI/UX 렌더링 검수
HTML 문항이 다양한 디바이스·브라우저에서 정상 렌더링되는지, 인터랙션이 올바르게 동작하는지 확인합니다.
04
채점 로직 검증
정답/오답/부분점수 시나리오별로 채점 로직이 정확하게 작동하는지 테스트합니다.
05
판정 (승인/반려)
모든 검수 기준 통과 시 승인, 이슈 발견 시 상세 피드백과 함께 제작팀으로 반려합니다.
산출물
review_report_{id}.json
{ status: "approved" | "rejected",
  issues[], feedback, reviewer, timestamp }
정합성 검증 크로스브라우징 채점 테스트 승인/반려
↩ 반려 시 제작팀으로 피드백 전달
TEAM 04
데이터팀
xAPI 적용 · interface 요구사항 구현

프로세스

01
xAPI Statement 설계
Actor-Verb-Object 구조로 학습 이벤트를 정의합니다. (attempted, answered, completed, passed, failed 등)
02
xAPI 이벤트 삽입
HTML 문항 내 사용자 인터랙션 포인트에 xAPI statement 발송 코드를 삽입합니다. (문항 시작, 응답, 제출, 완료)
03
Interface 통신 구현
LMS/LRS와의 통신 인터페이스를 구현합니다. postMessage, API 콜, 초기화/종료 핸들러 등을 적용합니다.
04
데이터 매핑 검증
메타정보의 성취기준·난이도가 xAPI context extensions에 올바르게 매핑되었는지 확인합니다.
산출물
item_{id}_xapi.html
xAPI statement 및 interface 통신이
내장된 최종 HTML 문항
xAPI 1.0.3 Actor-Verb-Object LRS 통신 postMessage Context Extensions
TEAM 05
플랫폼팀
xAPI 이벤트 · 통신 검증

프로세스

01
xAPI 이벤트 발화 테스트
문항 라이프사이클(시작→응답→제출→완료)에 따라 올바른 xAPI statement가 올바른 시점에 발송되는지 검증합니다.
02
Statement 구조 검증
발송된 xAPI statement의 JSON 구조가 xAPI 1.0.3 스펙에 부합하는지, 필수 필드 누락은 없는지 검증합니다.
03
Interface 통신 테스트
LMS ↔ 콘텐츠 간 postMessage 통신, 초기화 시퀀스, 종료 핸들링, 에러 처리가 정상 동작하는지 확인합니다.
04
LRS 데이터 적재 확인
실제 LRS에 데이터가 정상 적재되는지, 쿼리 시 기대한 결과가 반환되는지 end-to-end 테스트를 수행합니다.
05
판정 (승인/반려)
모든 검증 통과 시 최종 승인, 이슈 발견 시 상세 로그와 함께 데이터팀으로 반려합니다.
산출물
validation_report_{id}.json
{ xapi_events: ✅, interface: ✅,
  lrs_verified: ✅, status: "deployed" }
이벤트 발화 검증 스펙 준수 E2E 테스트 LRS 적재
↩ 반려 시 데이터팀으로 피드백 전달
