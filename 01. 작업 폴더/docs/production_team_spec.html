제작팀 에이전트 상세 설계

기획팀 메타정보(JSON)를 입력받아, LLM이 직접 HTML 문항을 생성하는 에이전트. 문항 1건 = 폴더(index.html + css/ + js/) 단위로 패키징.

문항유형 10 종
저작방식 LLM 직접 생성
패키징 폴더 단위
멀티미디어 이미지·수식·좌표
입력 스키마 FROM 기획팀

기획팀 에이전트가 출력한 item_meta_{id}.json이 제작팀의 유일한 입력입니다. 이 JSON 하나로 문항의 모든 것을 결정합니다.

문항 유형 카탈로그 10 TYPES

제작팀이 생성할 수 있는 전체 문항 유형입니다. 기본형 4종 + 인터랙티브형 3종 + 멀티미디어형 3종으로 구성됩니다.

MEDIA

MathML 또는 KaTeX로 수식을 렌더링하는 문항. 모든 기본형+인터랙티브형과 결합 가능.

SVG 인라인 이미지를 포함하는 문항. 도형, 그래프, 다이어그램 등. 외부 리소스 없이 자체 내장.

SVG 좌표평면에서 점 찍기, 직선 그리기, 그래프 조작 등 인터랙티브 수학 문항.

📌 결합 규칙: 멀티미디어 3종은 단독 유형이 아니라 기본형·인터랙티브형과 결합됩니다. 예: multiple_choice + MATH = 수식이 포함된 객관식, drag_and_drop + COORD = 좌표평면 드래그 문항.
저작 파이프라인 LLM DIRECT

LLM이 메타정보를 읽고 HTML/CSS/JS를 직접 생성합니다. 템플릿 없이 매번 새로 저작하되, 구조·스타일 가이드라인을 프롬프트로 제어합니다.

STEP 1. 메타정보 파싱 및 유형 결정 전처리

기획팀 JSON에서 item_type, content_area, authoring_guide를 추출하여, LLM 프롬프트에 주입할 컨텍스트를 구성합니다.

📋 유형 라우팅
  • item_type → 기본 HTML 구조 결정 (라디오, 드래그, textarea 등)
  • content_area = "도형과 측정" → SVG 이미지 생성 플래그 ON
  • content_area = "수와 연산" + 키워드 "함수" → KaTeX 수식 플래그 ON
  • authoring_guide.media_suggestion 존재 시 → 해당 미디어 유형 활성화

STEP 2. LLM 콘텐츠 생성 (1st pass) LLM

메타정보 + 스타일 가이드라인 + 유형별 구조 규칙을 포함한 프롬프트로 LLM을 호출하여 문항 콘텐츠(지문, 발문, 보기, 정답, 해설)를 생성합니다.

🤖 LLM 생성 범위
  • 지문(passage): 학습 맥락을 제공하는 텍스트 또는 상황 설명
  • 발문(stem): 학생에게 묻는 질문. Bloom's 수준에 맞는 동사 사용
  • 보기/선택지: 정답 1개 + 오답 N개. 오답은 전형적 오개념 기반 생성
  • 정답 데이터: 자동 채점용 정답 값 (문자열, 배열, 좌표 등)
  • 해설: 정답 근거, 풀이 과정, 오답 분석
  • 미디어: 필요 시 인라인 SVG (도형, 좌표평면, 그래프)

STEP 3. LLM HTML 코드 생성 (2nd pass) LLM

1st pass 콘텐츠를 HTML/CSS/JS로 변환합니다. 유형별 인터랙션 코드, 채점 로직, 타이머, 접근성을 포함한 완전한 코드를 생성합니다.

📦 생성 파일 3종
  • index.html: 문항 마크업 + 메타 배지 + 헤더 + 본문 + 푸터
  • css/style.css: 문항 유형별 스타일 + 반응형 + 다크모드 미지원(학습용이므로 라이트 고정)
  • js/item.js: 인터랙션(클릭, 드래그, 입력) + 채점 로직 + 타이머 + 피드백 표시
⚡ 2-pass 이유: 1회 호출로 콘텐츠+코드를 동시에 생성하면 코드 품질이 떨어집니다. 1st pass에서 교육적 콘텐츠 품질에 집중하고, 2nd pass에서 코드 구현에 집중하는 분리가 핵심입니다.

STEP 4. 코드 검증 및 보정 검증

생성된 HTML/CSS/JS의 구조적 무결성을 자동 검증합니다.

✅ 자동 검증 항목
  • HTML 파싱 가능 여부 (태그 닫힘, 중첩 오류)
  • 정답 데이터가 JS에 올바르게 내장되었는지
  • 필수 요소 존재: feedback 영역, 제출 버튼, 타이머
  • 접근성: ARIA 라벨, 키보드 내비게이션, 포커스 관리
  • CSS 변수 일관성, JS 문법 오류 없음
  • 인터랙티브 유형: 이벤트 핸들러 바인딩 확인

STEP 5. 폴더 패키징 및 출력 출력

검증 통과 시 3개 파일을 폴더로 패키징하여 출력합니다. meta.json도 함께 복사하여 4파일 구성.

산출물 구조 FOLDER

문항 1건 = 폴더 1개. 브라우저에서 index.html만 열면 모든 리소스가 로컬에서 동작합니다.

index.html 표준 구조

HTML STRUCTURE <!DOCTYPE html> <html lang="ko"> <head> <!-- 메타, 폰트, CSS 링크 --> <link rel="stylesheet" href="css/style.css"/> <!-- 수식 필요 시: KaTeX CDN --> </head> <body> <div class="item-container"> <!-- ① 헤더: 메타 배지 + 경로 + 성취기준 --> <header class="item-header"> [문항유형] [난이도] [배점] [제한시간] [depth_path] [성취기준 코드 + 내용] </header> <!-- ② 본문: 지문 + 발문 + 인터랙션 영역 --> <main class="item-body"> <div class="passage">지문/상황 텍스트 (선택적)</div> <div class="stem">발문 (질문)</div> <div class="interaction" role="group"> <!-- 유형별 인터랙션 UI --> </div> <div class="feedback" id="feedback"></div> </main> <!-- ③ 푸터: 제출/초기화 + 타이머 --> <footer class="item-footer"> [제출] [초기화] [⏱ 타이머] </footer> </div> <script src="js/item.js"></script> </body></html>

item.js 표준 인터페이스

JS INTERFACE // ── 모든 문항 유형이 구현해야 하는 표준 인터페이스 ── var ITEM = { answer: "...", // 정답 (string|array|object) type: "multiple_choice",// 문항 유형 score: 3, // 배점 timeLimit: 90, // 제한시간(초) submitted: false // 제출 상태 }; // 필수 함수 function getResponse() {} // 현재 사용자 응답 반환 function submitAnswer() {} // 채점 실행 + 피드백 표시 function resetItem() {} // 초기 상태로 복원 function getResult() {} // 채점 결과 객체 반환 // getResult() 반환 형태 (→ 데이터팀 xAPI 연동용) { correct: true, score: 3, maxScore: 3, response: "2", // 사용자 응답 원본 duration: 45, // 소요시간(초) timestamp: "..." // ISO 타임스탬프 }
🔗 다음 팀 연결: getResult()의 반환값이 데이터팀(TEAM 04)의 xAPI statement 생성 입력이 됩니다. 이 인터페이스가 팀 간 계약(contract)입니다.
LLM 프롬프트 설계 2-PASS

제작팀 에이전트의 핵심은 프롬프트 엔지니어링입니다. 2-pass 구조로 콘텐츠 품질과 코드 품질을 분리합니다.

1️⃣

1st Pass — 콘텐츠 생성 프롬프트

PASS 1: CONTENT system: "너는 2022 개정 교육과정 수학 교과 문항 출제 전문가이다. 주어진 메타정보를 기반으로 교육적으로 우수한 문항을 설계하라. 오답은 학생의 전형적 오개념에 기반하여 매력적으로 구성하라." user: "다음 메타정보를 기반으로 문항 콘텐츠를 JSON으로 생성하라. [메타정보] - 문항유형: {item_type} ({item_type_detail}) - 내용체계: {content_area} > {content_level1} > {content_level2} - 성취기준: {achievement_code} {achievement_desc} - 내용요소: {content_element} - Bloom's: {blooms_level} - 난이도: {difficulty} (score: {difficulty_score}) - 배점: {score}점 / 제한시간: {time_limit_sec}초 - 교급/학년: {school_level} {grade}학년 {semester}학기 [출제 가이드] {authoring_guide.direction} [제약 조건] {authoring_guide.constraints} [미디어 권장] {authoring_guide.media_suggestion} JSON 응답 형식: { passage: string|null, stem: string, options: [{label,text,isCorrect}]|null, answer: string|array|object, explanation: string, media: {type:'svg'|'katex'|null, content:string}|null, accessibility: {ariaLabel:string} }"
2️⃣

2nd Pass — 코드 생성 프롬프트

PASS 2: CODE system: "너는 교육용 웹 콘텐츠 프론트엔드 개발자이다. 주어진 문항 콘텐츠를 HTML/CSS/JS 3파일로 변환하라. 다음 규칙을 반드시 준수하라: [HTML 규칙] - 시맨틱 태그 사용 (header, main, footer, section) - ARIA 속성 포함 (role, aria-label, aria-live) - 외부 리소스 최소화 (폰트: Google Fonts, 수식: KaTeX CDN만 허용) - 인라인 SVG로 이미지 내장 (외부 이미지 파일 금지) [CSS 규칙] - CSS 변수로 색상 관리 (--primary, --success, --danger 등) - 모바일 반응형 (max-width: 480px) - font-family: 'Noto Sans KR', sans-serif - 기본 폰트 크기 16px 기준 [JS 규칙] - 표준 인터페이스 구현: ITEM 객체, getResponse, submitAnswer, resetItem, getResult - Vanilla JS만 사용 (jQuery 등 라이브러리 금지) - 터치 이벤트 지원 (드래그앤드롭 유형) - 타이머 구현 (시작/정지/표시) - 채점 결과를 window.postMessage로 부모에 전송 가능하게" user: "다음 문항 콘텐츠를 3개 파일로 코드 생성하라. [콘텐츠 JSON] {1st_pass_result} [메타정보] {원본 메타정보 요약} 3개 파일을 각각 ```html ```, ```css ```, ```js ``` 블록으로 분리하여 출력하라."
🔧 프롬프트 버전 관리: 프롬프트는 prompt_v{N}.md로 버전 관리하며, 문항 품질 이슈 발생 시 프롬프트를 수정하고 버전을 올립니다. meta.json의 source.prompt_version에 사용된 프롬프트 버전을 기록합니다.

G2. 수식입력기 MathType 적용 가이드

KaTeX 대체

수식 입력이 필요한 문항에서 KaTeX 대신 MathType(common-mathtype.js)을 사용합니다. 컨테이너가 JS를 로딩하며, 콘텐츠는 div 기반 수식 입력 영역만 제공합니다.

🔧 적용 규칙
  • 수식 입력 UI: <input> 대신 <div id="wiris{N}" onclick="showMathType(this.id, this.innerHTML, toolbar)"> 사용
  • 교급별 toolbar: 초등 'elementary' / 고등 'high' / 디폴트 중등. meta.json의 school_level로 자동 결정
  • 필기도구: showHandType(id, val, toolbar)로 필기 입력 모드 오픈 (디폴트)
  • 저장/닫기: confirmMathType()(저장+닫기), closeMathType()(저장 안 하고 닫기)
  • Textarea 모드: contenteditable="true" div 필수, textarea 태그 사용 불가
  • 필수 DOM: <div id="editorContainer"></div> 수식입력기 영역
  • CSS: #editorContainer 스타일 (position:absolute, z-index:1000, 720x200px)
⚡ v1 변경: 기존 설계의 "KaTeX CDN" → MathType(common-mathtype.js)로 전면 교체. 수식 렌더링이 아니라 수식 입력 UI를 제공하는 것이므로, LLM이 생성하는 코드에서 KaTeX 관련 CDN/auto-render를 제거하고 MathType div 구조로 대체합니다.
