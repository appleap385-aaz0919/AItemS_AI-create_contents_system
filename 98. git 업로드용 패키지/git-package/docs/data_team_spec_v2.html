데이터팀 에이전트 상세 설계 v2 — OctoPlayer 규격 반영

OctoPlayer 뷰어-콘텐츠 연동 가이드와 동아출판 콘텐츠 전송 데이터 정의를 기반으로, 검수 통과 HTML 문항에 커스텀 xAPI 로그 + postMessage 통신을 적용하는 에이전트.

연동 OctoPlayer
통신 postMessage
문항 Verb 7 종
데이터 restore + xAPI
메시지 sendLogToContainer
3레이어 아키텍처 OctoPlayer

OctoPlayer는 컨테이너(LMS) ↔ 뷰어(OctoPlayer) ↔ 콘텐츠(문항 iframe) 3레이어로 동작합니다. 우리 문항은 '콘텐츠' 레이어에 위치합니다.

🏗️

통신 흐름도

컨테이너
LMS/학습 플랫폼. 뷰어를 호스팅하고, xAPI 로그를 LRS로 전달하며, 수업 종료 등을 제어합니다.
↕ loadPlayer() / playerLoaded() / receiveLogFromViewer()
뷰어
OctoPlayer. 콘텐츠를 iframe으로 로딩하고, 콘텐츠↔컨테이너 간 메시지를 중계합니다. sendRestoreData, sendUserId, terminated 등을 콘텐츠에 전달합니다.
↕ postMessage (sendLogToContainer / sendRestoreData / nextPage / terminated)
콘텐츠 ★
우리 문항이 위치하는 레이어. iframe 내에서 동작하며, 뷰어에 postMessage로 xAPI 로그 + restore 데이터를 전송합니다.
🔑 핵심 변경: v1에서는 콘텐츠가 LRS에 직접 Statement를 전송하는 구조였지만, 실제 규격에서는 콘텐츠 → 뷰어(OctoPlayer) → 컨테이너(LMS) → LRS 순서로 중계됩니다. 콘텐츠는 window.parent.postMessage()만 사용합니다.

sendLogToContainer 표준 페이로드

핵심
sendLogToContainer PAYLOAD // 모든 이벤트에서 이 구조로 전송 const message = { type: 'sendLogToContainer', data: { "contentId": "IFRAME-001", // sendRestoreData에서 받은 값 그대로 "contentTag": { /* 뷰어에서 받은 객체 그대로 */ }, "timestamp": "2026-03-24T09:05:30.033Z", // 공통: 로그 발생 시간 "restore": { // 사용자 입력 상태 복원용 데이터 (JSON) }, "xAPI": { "verb": "completed", "object": { "id": "contentId", // lcms/conts-id + stage-num "type": "..." // 해당 시에만 (video, audio, external-link 등) }, "result": { "response": { /* 모듈별 응답 데이터 */ }, "duration": "PT45S", // ISO 8601 "success": true, "extensions": { // 이벤트별 상세 데이터 } }, "context": { "extensions": { "lcms/conts-id": "AAA-0001" // 컨테이너에서 전달받은 콘텐츠ID } } } } }; window.parent.postMessage(message, '*');
🔑 v1 대비 핵심 변경: 표준 xAPI 1.0.3 Actor-Verb-Object 구조가 아니라, 커스텀 경량 xAPI 포맷입니다. Actor 없음(userId는 뷰어가 관리), Verb는 문자열, Object는 간소화. restore와 xAPI가 하나의 메시지에 동시 포함됩니다.
xAPI 데이터 구조 커스텀 경량

동아출판 콘텐츠 전송 데이터 정의에 따른 커스텀 xAPI 구조입니다. 표준 xAPI 1.0.3과 다른 경량 포맷을 사용합니다.

response 구조 (채점/완료 시 전송)

response 구조 // xAPI.result.response — 채점/완료 시에만 포함 // 각 모듈(문항 유형)별로 사용자 응답을 CSS_ID 기준으로 구조화 { "모듈1": { "gap1": 1, // 객관식: 선택 번호 "gap2": 234 // 빈칸: 입력 값 }, "모듈2": { // 드래그: 배치 좌표 // 순서배열: 정렬 순서 // 매칭: 연결 쌍 } }
💡 response vs restore: response는 채점·완료 시에만 보내는 간략화된 데이터, restore는 매 이벤트마다 보내는 전체 복원용 데이터입니다. response는 restore의 간단한 버전입니다.
Verb 카탈로그 7 VERBS

웹문서 콘텐츠(문항) 유형에서 발생하는 7개 Verb입니다. v1의 표준 xAPI Verb와 다릅니다.

started

→

started 페이지 인 v1 attempted → started

콘텐츠가 로드된 시점. xAPI 하위에 verb만 발생 (result 없음).

발화: window.onload 완료 후 | 포함: { xAPI: { verb: "started" } }

presented 완료하기 버튼 클릭 신규

채점 없이 활동만 하는 콘텐츠의 완료. response + extensions(req-act-cnt, com-act-cnt, com-act-rt) 포함.

발화: 완료/제출 버튼 클릭 (정오 판별 없는 활동형 문항) | 포함: restore + xAPI.result

completed 채점하기 버튼 클릭 v1 answered+completed+passed/failed → completed 통합

정오가 있는 문항의 채점 완료. response + duration + success + extensions(skip, crt-rt, crt-cnt, incrt-cnt, req-act-cnt, com-act-cnt, com-act-rt) 포함.

발화: 채점/제출 버튼 클릭 (정오 판별 가능 문항) | 포함: restore + xAPI.result (가장 많은 데이터)

viewed 해설 보기

해설 보기 버튼을 클릭한 시점. verb만 발생 (result 없음).

발화: 해설 버튼 클릭 | 포함: { xAPI: { verb: "viewed" } }

reset 다시하기 버튼 클릭

다시하기 버튼 클릭 시점. verb만 발생 + restore 데이터 포함.

발화: 다시하기/초기화 버튼 클릭 | 포함: restore + { xAPI: { verb: "reset" } }

clicked 외부 링크 클릭 신규

콘텐츠 내 외부 링크 클릭 시. object.type:"external-link" + extensions.link-type 포함.

발화: 외부 링크 버튼 클릭 | 포함: xAPI.object + extrLink

left 페이지 아웃 v1 없음 → 추가

콘텐츠에서 정상 이탈(종료 버튼, 뷰어 terminated 수신 시). duration(체류시간) + restore 포함.

발화: terminated 메시지 수신 또는 GNB 이탈 | 포함: restore + xAPI.result.duration

적용 파이프라인 5 STEPS

검수 통과 폴더에 OctoPlayer 연동 코드를 비침습적으로 삽입합니다.

STEP 1. 메타정보 분석 + getResult() 매핑 분석

meta.json에서 item_type을 분석하여 액티비티 카운트 기준을 결정합니다. item.js의 getResult() 반환값을 동아출판 extensions 필드로 매핑합니다.

📋 getResult() → 동아출판 extensions 매핑
  • getResult().correct → success (crt-rt==1.0이면 true)
  • getResult().score / maxScore → crt-rt (소수 둘째자리 버림)
  • getResult().response → response (CSS_ID 구조로 변환)
  • getResult().duration → duration (ISO 8601: "PT{n}S")
  • 문항 유형별 → req-act-cnt (빈칸수, 드래그항목수 등)

STEP 2. octo-bridge.js 생성 생성

OctoPlayer 연동 전체를 담당하는 단일 브릿지 모듈을 생성합니다. 초기화, 이벤트 감지, xAPI 조립, postMessage 발송, 종료 처리를 모두 포함합니다.

octo-bridge.js 핵심 구조 var OCTO = { contentId: null, contentTag: null, userId: null, startTime: null, // 초기화 init: function() { this.requestUserId(); this.requestRestoreData(); this.listenMessages(); this.startTime = new Date(); }, // 뷰어 메시지 수신 listenMessages: function() { window.addEventListener('message', function(e) { if(e.data.type === 'sendRestoreData') { /* contentId, contentTag 저장, 복원 */ } if(e.data.type === 'sendUserId') { /* userId 저장 */ } if(e.data.type === 'terminated') { OCTO.sendLeft(); OCTO.deinit(); } if(e.data.type === 'nextPage') { /* 페이지 이동 */ } if(e.data.type === 'prevPage') { /* 페이지 이동 */ } }); }, // 7개 Verb 발송 함수 sendStarted: function() { /* verb만 */ }, sendPresented: function() { /* 활동형 완료 */ }, sendCompleted: function() { /* 채점형 완료 — getResult() 호출 */ }, sendViewed: function() { /* 해설 보기 */ }, sendReset: function() { /* 다시하기 */ }, sendClicked: function(linkType) { /* 외부 링크 */ }, sendLeft: function() { /* 이탈 — duration 포함 */ }, // 공통: 메시지 조립 + 발송 send: function(data) { data.contentId = this.contentId; data.contentTag = this.contentTag; data.timestamp = new Date().toISOString(); window.parent.postMessage({ type: 'sendLogToContainer', data: data }, '*'); }, // 종료 처리 deinit: function() { window.parent.postMessage({ type: 'iframeDeinit' }, '*'); } };

STEP 3. item.js 이벤트 후킹 후킹

제작팀 item.js를 수정하지 않고, submitAnswer/resetItem 함수를 래핑하여 xAPI 이벤트를 자동 발화합니다.

함수 래핑 패턴 // 원본 함수 보존 후 래핑 var _origSubmit = window.submitAnswer; window.submitAnswer = function() { _origSubmit(); // 원본 실행 // 채점형 문항이면 completed, 활동형이면 presented if (ITEM.type === 'essay' || !getResult().hasOwnProperty('correct')) OCTO.sendPresented(); else OCTO.sendCompleted(); }; var _origReset = window.resetItem; window.resetItem = function() { _origReset(); OCTO.sendReset(); };
✅ 비침습 유지: item.js 파일 자체는 수정하지 않습니다. octo-bridge.js가 로드된 후 전역 함수를 래핑하는 방식으로, 원본 코드의 해시값이 변하지 않습니다.

STEP 4. restore 데이터 생성 로직 생성

매 이벤트마다 현재 사용자 입력 상태를 JSON으로 직렬화하여 restore 객체를 구성합니다. 뷰어가 다시 sendRestoreData로 돌려주면 이전 상태를 복원합니다.

STEP 5. index.html 수정 + 패키징 출력

index.html의 item.js 아래에 octo-bridge.js 스크립트 태그 1줄을 추가합니다.
